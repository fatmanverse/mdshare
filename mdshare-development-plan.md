# Markdown 发布客户端开发任务拆解

## 1. 目标

这份文档把 MVP 拆成可以直接执行的开发任务，默认按一个人独立完成来设计。

建议节奏：

- 总周期：`3-4 周`
- 发布目标：`v0.1.0`
- 首版范围：编辑、预览、`HTML/PDF` 导出、本地文件管理

## 2. 里程碑总览

### Milestone 0：项目初始化

目标：

- 搭好可运行的桌面工程

交付结果：

- `Electron + React + Vite` 工程可启动

## 4. 推荐开发顺序

最稳妥的顺序：

1. 壳子跑起来
2. 编辑器能输入
3. 预览能渲染
4. 文件能保存
5. HTML 能导出
6. PDF 能导出
7. 细节体验和发布

不要一开始就做：

- 多格式导出
- 多文档标签页
- 插件系统
- 云同步

## 5. 每周计划建议

### 第 1 周

- 完成工程初始化
- 完成主界面与编辑器
- 完成基础 Markdown 预览

### 第 2 周

- 完成文件打开/保存
- 完成自动保存与恢复
- 完成 HTML 导出

### 第 3 周

- 完成 PDF 导出
- 完成最近文件
- 完成主题和目录

### 第 4 周

- 补测试
- 补 README
- 打包并发布首版

## 6. 验收清单

发布 `v0.1.0` 前至少确认：

- [ ] 能新建文档
- [ ] 能打开 `.md`
- [ ] 能保存和另存为
- [ ] 能实时预览
- [ ] 能切换主题
- [ ] 能导出 HTML
- [ ] 能导出 PDF
- [ ] 能恢复草稿
- [ ] 能查看最近文件
- [ ] 主流程没有明显卡顿

## 7. 首版之后的 Backlog

推荐下一阶段再做：

- [ ] `DOCX` 导出
- [ ] Front Matter 支持
- [ ] 自定义导出模板
- [ ] 图片资源打包策略
- [ ] 多页面导出
- [ ] 命令行导出

## 8. 风险提醒

需要特别留意：

- PDF 分页效果不要在首版追求完美
- 图片路径处理一定要早做验证
- 预览和导出必须共用一套渲染逻辑
- 文档边界要守住，不要做成笔记软件

## 9. 最终建议

如果你是一个人做，这个项目最关键的是节奏控制：

- 先做能用
- 再做好看
- 最后做扩展

一句话执行策略：

先把 `HTML` 导出打磨好，再把 `PDF` 做到“够稳定可发人”，首版就成立了。
